    Introduction

    Everperform Open API

    API Version 1

    API Domain api.everperform.com

    All requests must use https

    The Everperform API uses JSON:API spec

    Authentication

    curl "api_endpoint_here"
  -H "Authorization: Bearer <your_token>"

    Make sure to replace <your_token> with your API key.

    Everperform uses API keys to allow access to the API. You can acquire your API key from API Settings.

    Everperform expects the API key to be included in all API requests to the server in a header that looks like the following:

    Authorization: Bearer <your_token>

    You must replace <your_token> with your API key.

    Examples

    Single Object Response

    curl '/api/v1/object_type/object_id'
  -H "Authorization: Bearer <your_token>"

    single object requests are returned in the format

    {
  "data": {
    "id": "2",
    "type": "object_type",
    "attributes": {
      "attribute_name": "attribute_value"
    },
    "relationships": {
      "associated_object": {
        "data": {
          "id": "associated_object id",
          "type": "associated_object type"
        }
      },
    }
  }
}

    Root Object

    Parameter Type Description
    data ResourceObject JSON:API Resource Object, see Resource Objects

    Data Object

    Parameter Type Description
    id Integer, String Id of the object
    type String Object type
    attributes Object List of attributes for the object and its values
    relationships RelationshipsObject JSON:API Relationships Object, see Relationships Objects

    Object Collection Response

    object collection requests are returned in the format

    {
  "meta": {
    "total": 2,
    "page": 1,
    "per_page": 20
  },
  "data": [{
    "id": "1",
    "type": "object_type",
    "attributes": {
      "attribute_name": "attribute_value"
    },
    "relationships": {
      "associated_object": {
        "data": {
          "id": "associated_object id",
          "type": "associated_object type"
        }
      },
    }
  },
  {
    "id": "2",
    "type": "object_type",
    "attributes": {
      "attribute_name": "attribute_value"
    },
    "relationships": {
      "associated_object": {
        "data": {
          "id": "associated_object id",
          "type": "associated_object type"
        }
      },
    }
  }]
}

    Root Object

    Parameter Type Description
    data Array<ResourceObject> array of JSON:API Resource Objects, see Resource Objects

    Data Array Item

    Parameter Type Description
    id Integer, String Id of the object
    type String Object type
    attributes Object List of attributes for the object and its values
    relationships RelationshipsObject JSON:API Relationships Object, see Relationships Objects

    POST, PUT and PATCH

    POST/PUT/PATCH request bodies should be structured like:

    curl '/api/v1/kpi/1'
  -H "Authorization: Bearer <your_token>"
  -d '{
        "data": {
          "id": "1",
          "type": "object_type",
          "attributes": {
            "attribute_name": "attribute_value"
          },
          "relationships": {
            "associated_object": {
              "data": {
                "id": "associated_object id",
                "type": "associated_object type"
              }
            },
          }
        }
      }'

    Root Object

    Parameter Type Description
    data ResourceObject JSON:API Resource Object, see Resource Objects

    Data Object

    Parameter Type Description
    id Integer, String Id of the object (may be inherited from request path)
    type String Object type (may be inherited from request path)
    attributes Object List of attributes for the object and its values
    relationships RelationshipsObject JSON:API Relationships Object, see Relationships Objects

    Object ID and Type

    The request path will overide the data/id and data/type fields where applicable

    For example: A request on the following path: /open/v1/kpi_series/1, the type and id are assumed to be kpi_series and 1 respectively

    Error Responses

    Error responses are structured like:

    {
  "errors": [{
    "status": "error_code",
    "detail": "error details"
  }]
}

    Root Object

    Parameter Type Description
    errors ErrorObject JSON:API Error Object, see Error Objects

    This endpoint creates a new KPI Series

    HTTP Request

    POST /api/v1/kpi_series/

    Body Parameters

    Parameter Type Description
    kpi_series KPISeries KPI Series Object

    Permitted Attributes

    Parameter Type Default Description Values
    name String required Name of the KPI Series
    description String Description of the KPI Series
    direction String up Determines whether higher values are good or bad up down
    frequency String required Frequency for the KPI Values days weeks months years
    aggregate_type String sum Determines how the values are aggregated for reports sum avg
    has_target Boolean true Show and allow target values to be used
    default_target Double Target to use if no target is provided when creating KPI Values
    symbol String Symbol that represents the KPI values
    symbol_label String Label to describe the Symbol
    symbol_prepend Boolean true Determine whether the symbol is attached to the start(true) or end(false) of the value

    Permitted Relationships

    Parameter Type Description
    owner user, team, organisation required Owner of this KPI Series
    responsible user, team, organisation Responsible entity of this KPI Series, Defaults to owner
    kpi kpi KPI to associate with this KPI Series
    data_category data_category Data Category to associate with this KPI Series
    kpi and frequency cannot be changed after creation

    This endpoint creates or updates a KPI Value.

    The kpi_date value is transformed to be the start of the date period based on the frequency of the KPI Series
    • '2018-05-05' -> '2018-05-01' for a monthly kpi
    • '2018-05-05' -> '2018-01-01' for a yearly kpi
    A new KPI Value will be created if no existing KPI Value for the kpi_date and KPI Series exists

    HTTP Request

    POST /api/v1/kpi_values

    POST /api/v1/kpi_series/<series_id>/kpi_values

    URL Parameters

    Parameter Type Description
    series_id Integer optional The id of the KPI Series

    Permitted Attributes

    Parameter Type Description
    actual Double 'Actual' Value for the period
    target Double 'Target' Value for the period, defaults to KPISeries#default_target if null and KPISeries#has_target is true
    kpi_date String required ISO8601/RFC3339 formatted date YYYY-MM-DD
    A KPI Value with both Actual and Target set to Null will be marked as deleted

    Permitted Relationships

    Parameter Type Description
    kpi_series kpi_series KPI Series that the KPI Value belongs to
    KPI Series relationship may be omitted when using the PUT/POST /open/v1/kpi_series/<series_id>/kpi_values routes
